Agent Directory

All 17 Specialist Agents

Each agent is a self-contained Python class extending BaseAgent. They receive shared state from LangGraph, execute their task using the LLM, and return a partial state update.

🎯
Supervisor Agent
Core / Orchestration
The top of the LangGraph hierarchy. Routes execution between all specialist agents using a deterministic plan-following strategy (no LLM call for normal operation) and a hard step-cap safety valve (MAX_STEPS=12) to prevent infinite loops.
  • Walks state["plan"] in order — no LLM needed in common case
  • Hard cap of 12 supervisor turns before force-finishing
  • Human-in-the-loop halt: pauses if approval_status = "pending"
  • LLM fallback only for off-plan / free-form tasks
  • Guards against LLM re-picking already-completed agents
📋
Planning Agent
Core / Orchestration
Runs first in every pipeline. Parses the user's natural-language task and uses the LLM to produce an ordered JSON list of specialist agents to invoke. The Supervisor then follows this plan deterministically.
  • Converts free-text task → ordered agent plan array
  • Knows a full "apply" flow: scout→rank→gap→resume→cover→app
  • Includes profile_intelligence if profile not yet built
  • Safe fallback: ["job_scout","job_ranking"] if LLM fails
👤
Profile Intelligence Agent
Core Pipeline
Analyzes resume text and external profiles (GitHub via MCP), extracts structured data using the LLM, persists it to the database, and indexes it into the per-user ChromaDB RAG store for retrieval by downstream agents.
  • Extracts: skills, experience, projects, education, certifications, achievements
  • Fetches live GitHub profile via MCP tool
  • Indexes resume + project docs into ChromaDB (user-scoped)
  • Persists structured JSON to UserProfile table
  • Up to 15,000 characters of combined profile text analyzed
🔍
Job Scout Agent
Core Pipeline
Generates 2–3 optimized search queries from the user's profile (skills, roles, location), then fans out across LinkedIn, Indeed, Naukri, and Wellfound via the job-board MCP tool. Deduplicates by URL before persisting.
  • LLM-generated high-signal search queries (not generic role names)
  • Searches LinkedIn, Indeed, Naukri, Wellfound
  • URL-based deduplication prevents duplicate DB records
  • Sets initial status="discovered" on all new Job rows
  • Returns raw_jobs list for downstream ranking
⭐
Job Ranking Agent
Core Pipeline
Scores every discovered job 0–100 using a weighted multi-factor algorithm (no LLM call required). Selects the highest-scoring job as current_job to drive the per-job sub-pipeline.
  • Skill match (40% weight) — set intersection on required vs. user skills
  • Experience match (15%) — seniority level vs. estimated years
  • Location match (15%) — exact / remote / mismatch scoring
  • Remote preference match (10%)
  • Salary match (10%) — posted range vs. min expectation
  • Company quality heuristic (10%)
🎓
Skill Gap Agent
Core Pipeline
Compares the current job's requirements against the user's profile and identifies missing skills, certifications, and technologies. Generates a prioritized learning roadmap saved to the SkillGap table.
  • Identifies missing_skills, missing_certifications, missing_technologies
  • Produces learning_recommendations with priority levels (high/medium/low)
  • Recommends resource types: course, doc, project, cert
  • Persists gap analysis to SkillGap table (per job)
  • Gap data flows into Interview Prep and Learning agents
📄
Resume Optimization Agent
Core Pipeline
Generates a tailored, ATS-optimized version of the user's resume for the current job. Uses RAG retrieval to ground the output in the user's actual past work. Calculates an ATS keyword coverage score.
  • RAG-grounded: top-4 relevant chunks retrieved from ChromaDB
  • Rewrites bullets with strong action verbs + job keywords
  • Output: markdown with Summary, Skills, Experience, Projects, Education
  • ATS score = keyword coverage % (required skills found in resume)
  • Truthful: only reorganizes and rephrases, never invents experience
✉️
Cover Letter Agent
Core Pipeline
Writes a personalized cover letter (under 350 words) referencing specific projects/experiences from the user's profile. Also generates three outreach message templates: recruiter, referral request, and networking.
  • Concise (<350 words), warm-professional tone
  • References 1–2 specific projects/experiences for personalization
  • Recruiter outreach message (LinkedIn-style)
  • Referral request message template
  • General networking message template
🔐
Application Agent
Human-in-the-Loop
Packages the tailored resume + cover letter + outreach messages into an Application record. Sets approval_status="pending" and halts the LangGraph graph — the pipeline NEVER auto-submits. User must explicitly approve via the UI.
  • NEVER submits applications automatically (critical safety requirement)
  • Sets requires_approval=True → Supervisor pauses to END
  • Recommends apply method: easy_apply / company_portal / email
  • Saves full package to Application table (resume, cover letter, outreach)
  • Updates job status to "awaiting_approval"
🎤
Interview Prep Agent
Core Pipeline
For shortlisted jobs, generates comprehensive interview preparation material including company research, technical questions, behavioral questions (STAR format), DSA roadmap, and a mock interview script.
  • Company research summary (culture, focus, recent direction)
  • 6–8 technical interview questions tailored to the job
  • 5 behavioral / STAR-format questions
  • DSA roadmap: 5–8 topics ordered by priority
  • Mock interview opening script for practice
📊
Tracking Agent
Core Pipeline
Updates job application status through the full lifecycle and computes dashboard analytics: total jobs, status breakdown, applied count, and application-to-interview conversion rate.
  • Status lifecycle: discovered→scored→resume_ready→awaiting_approval→approved→applied→assessment→interview→offer/rejected
  • Creates TrackingEvent audit trail on every status change
  • Computes conversion rate: interviews / applied applications
  • Dashboard data available via API for bar charts and tables
🐙
GitHub Enhancement Agent
Enhancement
Fetches the user's public GitHub repositories via the MCP tool and provides actionable improvement suggestions to make their profile more impressive to recruiters.
  • README improvement suggestions per repository
  • Missing documentation identification
  • Architecture diagram recommendations
  • Missing tests / test coverage gaps
  • Dockerization and CI/CD pipeline suggestions
🌐
Portfolio Enhancement Agent
Enhancement
Analyzes the user's portfolio website and suggests new project ideas, better descriptions focused on impact/metrics, SEO improvements, technical blog topics, and personal branding changes.
  • 2–3 new project ideas aligned to target roles
  • Better project descriptions (focus: impact + metrics)
  • SEO: meta tags, keywords, page title suggestions
  • 3 technical blog post topics to demonstrate expertise
  • Personal branding: tagline, about section angle
🤝
Networking Agent
Enhancement
Generates a LinkedIn search strategy to find recruiters, hiring managers, and alumni at the target company, plus personalized message templates for each persona type.
  • 4–6 LinkedIn search terms to find the right people
  • Recruiter outreach message template
  • Hiring manager / team lead message template
  • Alumni connection message template
  • Human-controlled: generates templates, user sends them
🔔
Notification Agent
Intelligence
Composes and sends daily summary notifications across Email, Telegram, and Discord. Reports: new jobs discovered, applications awaiting approval, and active interview processes.
  • Counts: new jobs discovered, pending approvals, active interviews
  • Channels: Email (SMTP), Telegram Bot, Discord Webhook
  • Saves Notification records to DB for history tracking
  • Reports "no channels configured" gracefully if env vars missing
🔄
Learning Agent
Self-Improvement
The reflection loop of the system. Analyzes rejection patterns across all rejected jobs, cross-references with skill gaps identified at application time, and produces updated learning recommendations to improve future match scores.
  • Queries all rejected jobs + their associated SkillGap records
  • Identifies top 3 recurring missing skills (by frequency)
  • Revised prioritized learning plan (3–5 items)
  • Job search strategy adjustment suggestions
  • Closes the agentic feedback loop over time
💰
Salary Intelligence Agent
Intelligence
Provides market salary range estimates for the current job's role/location/experience level, compares against the posted range, and generates 3–4 specific salary negotiation talking points tailored to the user's strengths.
  • Market salary estimate (clearly framed as estimate, not live data)
  • Comparison: posted salary vs. market estimate
  • 3–4 actionable negotiation talking points
  • Tailored to user's profile strengths and experience level

Deep Dive

How the Agents Work Together

Under the hood: state sharing, RAG retrieval, safety mechanisms, and the self-improvement feedback loop.

🧠 Shared State (LangGraph)

  • All agents communicate via a single CareerPilotState TypedDict passed through the graph
  • plan — ordered list of agents to run (set by Planning Agent)
  • profile — extracted user data (set by Profile Intelligence)
  • raw_jobs / scored_jobs — job lists flowing Scout→Ranking
  • current_job — best-scored job, drives the per-job sub-pipeline
  • tailored_resume / cover_letter — generated documents
  • requires_approval + approval_status — human checkpoint flags
  • logs — append-only execution log for UI display
  • scratch — agent-specific side data (analytics, suggestions)

🔍 RAG Architecture

  • Each user gets their own ChromaDB collection: user_{id}_docs
  • Profile Intelligence indexes resume text + each project as separate documents
  • Resume Optimization retrieves top-4 chunks using job title + required skills as the query
  • This grounds the LLM output in the user's actual real past work
  • Embeddings use the same provider as the LLM (Gemini or OpenAI)

🔐 Safety Architecture

  • Application Agent NEVER submits — only packages and requests approval
  • Supervisor detects requires_approval=True and routes to FINISH
  • LangGraph pauses at MemorySaver checkpoint until user decision
  • User approves via UI → POST /applications/{id}/decision
  • MAX_STEPS=12 prevents infinite supervisor loops
  • LLM fallback guards against re-picking completed agents

🔄 Self-Improvement Loop

  • Learning Agent queries all Job rows with status="rejected"
  • Joins with SkillGap records to find what skills were missing
  • Counts frequency of each missing skill across all rejections
  • LLM synthesizes: top 3 recurring gaps + revised learning plan + strategy adjustment
  • This creates a virtuous feedback loop improving match scores over time

📦 MCP Tool Layer

  • fetch_github_profile(url) — fetches public GitHub repos summary
  • search_jobs_external(query, location, source) — searches LinkedIn, Indeed, Naukri, Wellfound
  • send_notification(channel, title, body) — Email/Telegram/Discord
  • Functions have exact MCP signatures — swap mock bodies for real API calls without changing any agent code

🗃️ Memory Architecture

  • Short-term: CareerPilotState flows through the graph for one run
  • Long-term: SQLite/Postgres persists profiles, jobs, applications across runs
  • Semantic: ChromaDB vector store for RAG over personal documents
  • Checkpointed: LangGraph MemorySaver enables pause/resume for HITL
  • Observability: AgentRun table logs every agent invocation with timing and status

# Every agent inherits from BaseAgent class ResumeOptimizationAgent(BaseAgent): name = "resume_optimization" def run(self, state: Dict[str, Any]) -> Dict[str, Any]: # 1. Read from shared state profile = state.get("profile", {}) job = state.get("current_job", {}) # 2. RAG retrieval — grounded in real user docs store = PersonalDocStore(user_id=state["user_id"]) chunks = store.query(job["title"] + required_skills, n_results=4) # 3. LLM call with rich context tailored_resume = self.invoke_llm(RESUME_PROMPT.format(...)) # 4. Return partial state update return { "tailored_resume": tailored_resume, "logs": ["[resume_optimization] completed"], } # BaseAgent.__call__ wraps run() with: # - Timing measurement # - Error handling → status="error" in AgentRun table # - DB logging of every invocation (agent_runs table)
